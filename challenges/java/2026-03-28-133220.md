# Challenge: Thread‑Safe LRU Cache in Java

## Difficulty
Medium

## Description
Design and implement a **thread‑safe Least Recently Used (LRU) cache** in Java.  
The cache should store key‑value pairs, automatically evict the least‑recently accessed entry when it reaches its maximum capacity, and allow concurrent access from multiple threads without corrupting its internal state.

Your implementation should expose the following public API:

```java
public class LRUCache<K, V> {
    public LRUCache(int capacity) { … }

    public V get(K key) { … }

    public void put(K key, V value) { … }

    public int size() { … }

    public List<K> getKeysInOrder(); // most‑recently used first
}
```

- `get` returns the value associated with the key, or `null` if the key is not present. Accessing a key updates its recency.
- `put` inserts a new key‑value pair or updates an existing key’s value. Inserting a new entry may trigger eviction if the cache is full.
- `size` returns the current number of entries.
- `getKeysInOrder` returns a snapshot list of the keys ordered from most‑recently used to least‑recently used.

The cache must work correctly under heavy concurrent reads and writes.

## Requirements
- The cache must support generic key and value types (`K` and `V`).
- Capacity is fixed at construction time and cannot be changed later.
- All public methods must be **thread‑safe**.
- Eviction follows the LRU policy: the entry that has not been accessed for the longest time is removed first.
- Operations (`get`, `put`, `size`) should have an average‑case time complexity of **O(1)**.
- Do **not** use any third‑party libraries; only standard Java SE APIs are allowed.

## Bonus
- Provide a non‑blocking (lock‑free) implementation using `java.util.concurrent` primitives such as `ConcurrentHashMap` and `AtomicReference`.
- Add a method `public void clear()` that empties the cache safely while other threads may be accessing it.
- Implement a `remove(K key)` method that deletes a specific entry without affecting the recency order of the remaining items.
