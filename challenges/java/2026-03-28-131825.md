    # Challenge: Java Text Analyzer

## Difficulty
Medium

## Description
Write a Java program that reads a block of text from standard input and provides the following analyses:

1. **Word Count** – Total number of words in the text.
2. **Sentence Count** – Total number of sentences (a sentence ends with a period `.`, exclamation mark `!`, or question mark `?`).
3. **Most Frequent Word** – The word that appears most often (case‑insensitive). If there is a tie, return any one of the most frequent words.
4. **Average Word Length** – The average length of all words, rounded to two decimal places.

The program should handle punctuation gracefully (e.g., commas, semicolons, parentheses) and ignore them when counting words and determining frequency.

## Requirements
- Read the entire input until EOF (or until a blank line if you prefer).
- Treat words as sequences of alphabetic characters (`A‑Z`, `a‑z`). Numbers, symbols, and mixed alphanumeric tokens should be ignored.
- The output must be printed to the console in the following exact format:

```
Word Count: <number>
Sentence Count: <number>
Most Frequent Word: <word>
Average Word Length: <value>
```

- Use only standard Java libraries (no external dependencies).
- The program should be efficient enough to handle texts up to at least 10,000 words without noticeable delay.

## Bonus
- Add a command‑line argument that allows the user to specify a list of stop‑words (e.g., "the", "and", "of") that should be excluded from the most frequent word calculation.
- Include functionality to display the top 5 most frequent words along with their counts.
- Support Unicode letters (e.g., accented characters) as part of words.
