# Challenge: **Recursive Word Ladder**

## Difficulty
Medium

## Description
You are given two English words, `start` and `end`, that are of the same length, and a list of allowed intermediate words called the *dictionary*. A **word ladder** is a sequence of words that begins with `start`, ends with `end`, and where each successive word differs from the previous one by exactly one letter. All intermediate words must exist in the provided dictionary.

Your task is to write a function `find_word_ladder(start, end, dictionary)` that returns **any** shortest possible word ladder as a list of strings. If no ladder exists, return an empty list.

**Example**

```text
start = "hit"
end   = "cog"
dictionary = ["hot","dot","dog","lot","log","cog"]
```

One valid shortest ladder is:

```text
["hit", "hot", "dot", "dog", "cog"]
```

## Requirements
- The function must accept three parameters:
  1. `start` – a string containing only lower‑case letters.
  2. `end` – a string of the same length as `start`.
  3. `dictionary` – a list (or set) of strings, each the same length as `start`.
- All words are assumed to be lower‑case alphabetic characters.
- The returned ladder must start with `start` and end with `end`.
- Each adjacent pair of words in the ladder must differ by exactly one character.
- Every intermediate word (i.e., all words except `start` and `end`) must be present in `dictionary`.
- If multiple shortest ladders exist, returning any one of them is acceptable.
- If no ladder can be formed, return an empty list `[]`.

## Bonus
- **Optimized Performance:** Implement the solution with a time complexity better than *O(N²)* where *N* is the size of the dictionary (Hint: consider pre‑computing adjacency or using bidirectional search).
- **Case Insensitivity:** Allow the function to handle mixed‑case inputs by normalizing them internally.
- **Path Count:** Provide an additional function `count_word_ladders(start, end, dictionary)` that returns the total number of distinct shortest ladders (modulo `10⁹ + 7`). This function should run efficiently even for large dictionaries.
